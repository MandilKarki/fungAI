1. System-wide architecture

Every surface (CLI, gateway channels) drives the same Agent loop. Every aegis_sentinel persona is that same loop, just configured differently. Nothing here is a separate codebase — one class, many configurations.

2. Cybersecurity analyst agent — six independent personas

Each persona gets its own memory backend, its own skill catalog, and its own LLM provider by default — verified live (see chat). They only share data when the orchestrator deliberately passes the same memory object into a delegated call.

Independent by default

Call any build_X_agent(provider=...) with no memory= argument and it gets a fresh, private StateBackend(). Two SOC and vuln agents built this way share nothing — proven live in this session.

Sharing is opt-in

Pass the same memory object to two build_X_agent() calls (exactly what orchestrator.py does) and they now see each other's files — e.g. an escalated SOC alert becoming visible to incident_response.

Own skills, own model

Each domain builder now accepts its own skill_catalog= and its own provider= — a persona can run a different LLM and a completely separate skill library from every other persona.
